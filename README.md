# Smart Logistics Copilot | 智能物流全局调度系统

## 项目简介
本项目是一个基于大语言模型（DeepSeek V3）与真实路网拓扑分析（OSMnx）构建的智能物流路径规划与调度系统。系统能够将自然语言形式的调度指令（例如“某路段拥堵，请立即改道”）转化为可执行的图论算法逻辑，并在SaaS仪表盘中实时完成动态重排线与可视化展示。

## 核心功能演示

## 核心技术与特性

### 1. 语义级全局调度
系统接入 DeepSeek V3 API，彻底改变了传统的表单点选式调度操作。它能够直接解析非结构化的自然语言，精准提取业务人员口述的“拥堵路段”、“封控区域”等限制条件，并将其自动化转化为底层图节点和边权重的更新指令。

### 2. 真实空间拓扑分析
底层的路径计算不依赖于纯在线的地图路线规划API，而是利用OSMnx与NetworkX将真实的城市路网转化为严谨的数学图结构。这种设计允许我们在本地内存中进行低延迟的最短路径计算（如Dijkstra算法），并能够对任意一条街道的连通性进行细粒度控制。

### 3. 热数据分离与坐标适配 (Hot-Data Separation)
* **本地拓扑极速响应**：对于核心业务区域的高频调度，系统直接命中本地 Graph 缓存，实现毫秒级改道计算。
* **API 动态回退机制**：对于长尾的模糊地址，系统动态调用高德地图API进行地理编码补充精准度。
* **坐标系原生对齐**：底层算法在数据交互时，原生处理了高德的GCJ-02坐标系到国际WGS-84坐标系的偏移校准，确保地图图层渲染零误差。

### 4. 现代化交互仪表盘
前端基于Streamlit构建。在路线对比场景中，系统采用了视觉分层设计：废弃路线以半透明红保留，新计算的畅通路线以亮绿色高亮叠加，为决策者提供最直观的前后对比上下文。

## 核心文件结构

* `app.py`：系统交互主入口。负责构建前端控制台界面、处理用户输入状态流以及调用 PyDeck 完成复杂地图图层的渲染。
* `routing_engine.py`：路径规划引擎。封装了高德地理编码、坐标转换逻辑，以及基于NetworkX的本地图论路径搜索算法。
* `llm_parser.py`：意图解析模块。负责与 DeepSeek V3 的交互逻辑，通过精心设计的Prompt约束，确保自然语言被稳定解析为结构化的JSON路网干预参数。

## 快速启动指南

### 1. 环境依赖
请确保环境中已安装以下核心基础库（推荐 Python 3.9+）：
```bash
pip install streamlit pydeck pandas python-dotenv requests
# 核心算法库：
pip install osmnx networkx
```

### 2. 环境配置
出于安全考虑，本项目不包含 API 密钥。请在项目根目录下新建一个 `.env` 文件，并严格按照以下格式配置你的专属密钥：
```text
DEEPSEEK_API_KEY = "your_deepseek_api_key_here"
GAODE_API_KEY = "your_amap_api_key_here"
```

### 3. 启动系统
在终端中执行以下命令，即可在本地浏览器中唤醒智能调度工作台：
```bash
streamlit run app.py
```
